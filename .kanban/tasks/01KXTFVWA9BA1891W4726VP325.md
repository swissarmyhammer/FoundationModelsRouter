## What
Compaction quality evals on Apple's Evaluations framework (WWDC26), per compaction_plan.md §5: a new gated test target `Tests/FoundationModelsRouterEvals/` (add to `Package.swift`, gated on `FM_ROUTER_INTEGRATION_TESTS` like other real-model tests).

`CompactionEvaluation: Evaluation`:
- **Subject**: compact the sample's transcript with the prompt under test, resume a session over the result, ask the sample's question — return answer + `CompactionResult`.
- **Dataset**: 20–30 hand-written seed transcripts (`ArrayLoader<PlantedFacts>`) with varied lengths, tool traffic, and multiple planted facts in the transcript head ("the API key lives in `.env.example`", "we chose tabs over spaces"); questions answerable only from folded content. Scale later with SampleGenerator.
- **Evaluators**: mechanical `FactRetention` metric (answer contains the planted fact), mechanical `UnderTarget` metric (`tokensAfter <= targetTokens`), and a `ModelJudgeEvaluator` (judge ≥ as capable as subject) scoring Faithfulness and Continuability on a four-point scale.
- Test entry: `@Test(.evaluates(...))` asserting mean FactRetention ≥ 0.9 (see the sketch in compaction_plan.md §5).

The same evaluation pointed at different `CompactionPrompt`s (the segment records the prompt name) is the hill-climbing loop for the default prompt — structure the dataset/runner so a prompt name is a parameter.

## Implementation Notes (2026-07-24)

"Apple's Evaluations framework (WWDC26)" is a real, installed framework in this sandbox's Xcode-beta toolchain — it lives at `Developer/Platforms/MacOSX.platform/Developer/Library/Frameworks/Evaluations.framework` (a developer-only framework analogous to XCTest, not in the regular SDK frameworks path). SwiftPM's `.testTarget` automatically adds the required test-framework search path, so `Package.swift` needed only an ordinary `.testTarget` declaration — no unsafe flags.

The framework's real API (read from its `.swiftinterface`, not guessed from the plan's pseudocode) requires `Evaluation.Sample.ExpectedValue == Subject.Value`, so `CompactionEvaluationOutcome` is one `Codable` struct serving as both a sample's ground truth and a subject's produced result. `ModelJudgeEvaluator` requires `ModelSampleProtocol`, so the dataset uses Apple's own `ModelSample<CompactionEvaluationOutcome>` rather than a fully custom sample type.

24 hand-written fixtures (`CompactionEvalDataset.swift`/`CompactionEvalSeed.swift`) cover single- and multi-fact heads, tool-traffic variants, and varied lengths. `CompactionEvaluation.swift` implements the `Evaluation` conformer with `prompt`/`budget` as init parameters (the hill-climbing loop is literally constructing a second `CompactionEvaluation` value). The gated real-model path (`Support/CompactionEvalRealSubjectRunner.swift`) is a lazy-loading actor so the evaluation and its `.evaluates(...)` trait argument can be constructed synchronously while the real MLX model load is deferred to first actual use.
