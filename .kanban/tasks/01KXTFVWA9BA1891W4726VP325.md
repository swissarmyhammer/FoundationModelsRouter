## What
Compaction quality evals on Apple's Evaluations framework (WWDC26), per compaction_plan.md §5: a new gated test target `Tests/FoundationModelsRouterEvals/` (add to `Package.swift`, gated on `FM_ROUTER_INTEGRATION_TESTS` like other real-model tests).

`CompactionEvaluation: Evaluation`:
- **Subject**: compact the sample's transcript with the prompt under test, resume a session over the result, ask the sample's question — return answer + `CompactionResult`.
- **Dataset**: 20–30 hand-written seed transcripts (`ArrayLoader<PlantedFacts>`) with varied lengths, tool traffic, and multiple planted facts in the transcript head ("the API key lives in `.env.example`", "we chose tabs over spaces"); questions answerable only from folded content. Scale later with SampleGenerator.
- **Evaluators**: mechanical `FactRetention` metric (answer contains the planted fact), mechanical `UnderTarget` metric (`tokensAfter <= targetTokens`), and a `ModelJudgeEvaluator` (judge ≥ as capable as subject) scoring Faithfulness and Continuability on a four-point scale.
- Test entry: `@Test(.evaluates(...))` asserting mean FactRetention ≥ 0.9 (see the sketch in compaction_plan.md §5).

The same evaluation pointed at different `CompactionPrompt`s (the segment records the prompt name) is the hill-climbing loop for the default prompt — structure the dataset/runner so a prompt name is a parameter.

## Acceptance Criteria

- [ ] FactRetention, UnderTarget, and the two judged dimensions all report; the gated test asserts mean FactRetention ≥ 0.9
- [ ] Running the evaluation with two different prompt names yields per-prompt attributable results
